## What Is Emergent?

**Emergence:** Properties that arise from interactions but don't exist in the parts.

| Not Emergent (in the parts) | Emergent (from interactions) |
|-----------------------------|------------------------------|
| Lines of code | Working software |
| Individual commits | Project momentum |
| Documentation files | Shared understanding |
| Trap checklists | Intuition for traps |
| Git branches | Collaboration patterns |
| Token limits | Session rhythm |

**The 60%+ rule:** If more than 60% of the value comes from what emerges between the parts rather than from the parts themselves, the system is alive.
